> 基于小米 MiMo 多模态大模型构建的智能相册管理系统，实现照片自动分类、自然语言语义搜索、重复检测去重的全流程 AI 化。

## 痛点

iOS 用户将照片存入 iCloud Drive 后，只能通过"文件"App 查看，无法像系统相册一样按时间线浏览、按内容搜索、自动分类、去重。系统相册与云端照片完全割裂，用户面对数千张照片束手无策。

## 解决方案

我们构建了一套基于多模态 AI 的智能相册管理引擎，将非结构化的照片集合转化为可搜索、可分类、可去重的结构化知识库。

## 核心能力

| 模块 | 能力 | 技术实现 |
|------|------|----------|
| **AI 智能分类** | 自动识别照片内容（人物、风景、美食、文档、宠物等），生成多维度标签 | MiMo 多模态模型推理 |
| **语义搜索** | 支持"去年在海边拍的日落"级别中文自然语言查询 | 中文 NLU 解析 + 标签匹配引擎 |
| **OCR 文字提取** | 自动提取照片中的文字信息（截图、文档、票据） | 多模态视觉理解 |
| **重复检测** | 感知哈希 + 特征向量相似度，识别重复/高度相似照片 | pHash + Embedding 余弦相似度 |
| **时空浏览** | EXIF 元数据提取，按时间线 + 地图双维度组织照片 | EXIF 解析 + Leaflet 地图聚合 |
| **色彩分析** | 实时提取照片主色调，驱动动态 UI 配色 | Canvas 像素采样 + 色彩混合 |

## AI 工作流

```
照片导入 → 多模态模型推理 → 标签 + 向量生成 → 索引入库
    │            │                │              │
    │      ┌─────┴─────┐   ┌─────┴─────┐   ┌───┴───┐
    │      │ 内容识别   │   │ Embedding │   │ 向量库 │
    │      │ 场景分类   │   │ 特征提取  │   │ 索引   │
    │      │ OCR 提取   │   │ 相似度    │   │ 缓存   │
    │      └───────────┘   └───────────┘   └───────┘
    │
    ▼
用户查询("海边日落") → NLU 解析(时间+地点+内容) → 混合检索 → 排序返回
```

## 功能清单

- **Grid 相册**：CSS Grid 3 列布局，Material Design 3 风格，支持动态色彩提取
- **大图浏览**：Hero 动画 Container Transform，双指缩放，左右滑动切换
- **时间线**：按月份自动分组，照片数量统计
- **地图视图**：基于 EXIF GPS 数据的地图聚合展示，标记点聚合
- **分类导航**：AI 自动分类（人物/风景/文档/宠物/食物），支持自定义标签
- **语义搜索**：中文自然语言查询，支持时间+地点+内容组合筛选
- **去重引擎**：pHash 感知哈希 + 特征向量余弦相似度双引擎检测
- **批量操作**：多选、批量分享/下载/删除、AI 批量分析
- **编辑工具**：裁剪旋转、亮度/对比度/饱和度调节、滤镜、标注绘制
- **拼图合成功能**：多张照片自动排版，Canvas 渲染，一键下载
- **收藏与置顶**：右滑收藏，置顶优先展示
- **回收站**：软删除，支持恢复和批量清理
- **幻灯片**：Ken Burns 效果全屏放映
- **主题切换**：5 套 Material Design 3 配色方案
- **PIN 锁屏**：4 位数字密码 + 生物识别保护
- **PWA 支持**：离线可用，桌面安装

## 技术选型

| 层 | 技术 |
|------|------|
| 前端 | 单文件 HTML/CSS/JS（~8000 行），零依赖构建 |
| 设计系统 | Material Design 3 + 动态色彩 |
| AI 推理 | 多模态视觉模型（图像分类 + OCR + Embedding） |
| 向量检索 | FAISS / USearch（余弦相似度） |
| 去重 | pHash 感知哈希 + 特征向量 |
| 地图 | Leaflet.js + OpenStreetMap |
| 存储 | iCloud Drive（图片）+ LocalStorage（索引/状态） |
